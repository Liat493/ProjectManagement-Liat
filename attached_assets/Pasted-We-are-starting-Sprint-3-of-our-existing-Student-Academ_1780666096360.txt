Feature to implement:
Add the full "Risk Alerts" analytics module.

General Risk Alerts Requirements:
- The Risk Alerts module must be fully integrated with the existing dashboard.
- Alerts must appear in a dedicated Risk Alerts panel/widget on the dashboard.
- All generated alerts must be stored in the database.
- Each alert must include:
  - Alert ID
  - Student ID, if relevant to the existing schema
  - Alert Type
  - Course Name, if relevant
  - Alert Title
  - Alert Message
  - Severity Level: Low / Medium / High
  - Date Created
  - Status: Active / Resolved
  - Recommendation Text
  - Related User Story ID: US1–US7
- The module should support future scalability and additional alert types.
- The module should remain consistent with the existing academic analytics system.

User Story 1 – Real-Time Low Grade Alert:
- Define a configurable grade threshold.
- Generate an alert whenever a new grade falls below the threshold.
- Store low-grade alerts in the database.
- Display the alert immediately on the dashboard/Risk Alerts panel.
- Show course name, grade received, and recommended action.
- Alert severity should increase as grades become lower.
- If the system does not currently support live grade entry, treat “real-time” as immediate alert generation based on the existing loaded or updated grade data. Do not add a new grade entry UI.

User Story 2 – Attendance Risk Alert:
- Calculate attendance percentage per course.
- Define configurable attendance thresholds.
- Generate alerts when attendance drops below the threshold.
- Display the current attendance percentage.
- Show recommendations for improving attendance.

User Story 3 – Declining Performance Trend Alert:
- Analyze historical grades per course.
- Detect declining grade trends across multiple assessments.
- Generate alerts when a consistent downward trend is identified.
- Display trend information visually or clearly in text.
- Include recommendations for improvement.

User Story 4 – Missing or Late Submission Alert:
- Integrate assignment deadlines using existing assignment/submission data where available.
- Detect assignments not submitted by the due date.
- Detect late submissions.
- Generate alerts automatically.
- Display assignment name, due date, course name if available, and submission status.

User Story 5 – High-Risk Course Alert:
- Create a clear and simple risk scoring model.
- Combine grades, attendance, and submissions.
- Calculate a risk score per course.
- Classify courses into Low, Medium, and High Risk.
- Generate alerts for High Risk courses.
- Display the risk score and contributing factors.

User Story 6 – Alert History:
- Store all generated alerts.
- Create a dedicated Alert History page/screen.
- Allow sorting by:
  - Date
  - Severity
  - Course
  - Alert Type
- Allow filtering by alert status.
- Support pagination for future scalability if it can be implemented safely without affecting existing functionality.

User Story 7 – Recommendations with Alerts:
- Every alert must include a recommendation.
- Create recommendation rules for each alert type.
- Display recommendations clearly in the UI.
- Recommendations should be actionable and easy for a student to understand.
