Required fixes:

1. Risk Alerts – Add missing Declining Performance demo coverage
The Risk Alerts module is missing visible demo coverage for the Declining Performance Trend Alert.

Fix:
- Add or adjust realistic demo grade history for one existing demo student/course so that the existing Risk Alerts logic naturally generates a Declining Performance Trend Alert.
- The declining trend should be based on multiple assessments with a clear sustained decline.
- Do not hardcode the alert.
- Do not create a fake UI-only alert.
- Do not change unrelated Risk Alerts logic.
- Do not affect Grade Averages, Heatmap Analytics, Smart Recommendations, Learning Habit Tracking, or any existing feature.
- The new alert should appear in Risk Alerts and should be traceable to the relevant User Story for declining performance.

2. Risk Alerts – Remove duplicate alerts
There are duplicate or near-duplicate alerts, especially Low Grade alerts for the same course/grade/source item.

Fix:
- Prevent duplicate alerts for the same student, course, alert type, source item, date, and value.
- Example: the same Low Grade alert for Calculus II, same quiz, same grade, and same date should appear only once.
- Keep separate alerts only if they are based on genuinely different source items, such as different assignments, different grades, different dates, or different alert types.
- Do not remove valid unique alerts.
- Do not change the alert UI layout.
- Do not change unrelated Risk Alerts behavior.

3. Learning Habit Tracking – Remove “Missing” from Submission Habits
The Learning Habit Tracking module currently shows Submission Habits with On Time, Late, and Missing.

Fix:
- In Learning Habit Tracking only, remove the “Missing” category from the Submission Habits widget.
- Keep only:
  - On Time
  - Late
- Update the visual bar, labels, counts, and text so they reflect only On Time vs Late submission habits.
- Do not affect the existing separate Submission Rate feature.
- Do not remove missing assignment logic from other modules such as Dashboard, Risk Alerts, Smart Recommendations, or Submission Rate.
- This change applies only to the Submission Habits section inside Learning Habit Tracking.

4. Smart Recommendations – Remove duplicate or near-duplicate recommendations
Smart Recommendations currently shows some duplicate or near-duplicate recommendation cards.

Fix:
- Do not show multiple recommendation cards when they refer to the same student, course, recommendation type, source item, and reason.
- Keep separate recommendations only when they are based on genuinely different source items, such as:
  - different assignments
  - different alerts
  - different grades
  - different weak topics
  - different courses
- Do not remove valid unique recommendations.
- Do not change the Smart Recommendations page layout.
- Do not change recommendation logic beyond deduplication.
- Preserve dismiss and mark-completed behavior.

6. Dashboard – Display grade average as a grade, not a percentage
The Dashboard currently shows the grade average as a percentage, which is less clear for an academic grade average.

- Do not change Grade Averages page logic.
- Keep percentage signs for:
  - Submission Rate
  - Attendance / Attendance Rate
