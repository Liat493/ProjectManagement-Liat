Authentication API Endpoints:

Create or update backend routes:

1. POST /api/auth/signup

Request body:
{
  "fullName": "Alex Cohen",
  "email": "alex@example.com",
  "password": "123456",
  "confirmPassword": "123456",
  "semester": "Semester A"
}

Behavior:
- Validate input
- Check if email already exists
- Hash password
- Insert new user into users table
- Insert linked student into students table
- Optionally create default demo academic data for the new student
- Return user and student profile without password_hash

Response example:
{
  "success": true,
  "message": "Account created successfully",
  "user": {
    "id": 1,
    "fullName": "Alex Cohen",
    "email": "alex@example.com",
    "role": "student"
  },
  "student": {
    "id": 1,
    "userId": 1,
    "fullName": "Alex Cohen",
    "email": "alex@example.com",
    "semester": "Semester A"
  }
}

2. POST /api/auth/login

Request body:
{
  "email": "alex@example.com",
  "password": "123456"
}

Behavior:
- Validate input
- Find user by email
- Compare password with password_hash
- Get linked student profile
- Return user and student profile without password_hash

Response example:
{
  "success": true,
  "message": "Login successful",
  "user": {
    "id": 1,
    "fullName": "Alex Cohen",
    "email": "alex@example.com",
    "role": "student"
  },
  "student": {
    "id": 1,
    "userId": 1,
    "fullName": "Alex Cohen",
    "email": "alex@example.com",
    "semester": "Semester A"
  }
}

3. GET /api/auth/me/:userId

Purpose:
Used to restore logged-in user after refresh.

Behavior:
- Receive userId or token depending on implementation
- Return user and student profile if exists
- Do not return password_hash

4. POST /api/auth/logout
