Feature: Heatmap Analytics

Implement the following User Stories:

US1 – Attendance Heatmap
- Display student attendance data in a heatmap format.
- Each course should appear as a row.
- Time periods such as weeks or months should appear as columns.
- Attendance percentages should be visualized by intensity.
- Retrieve attendance data dynamically from the existing database/data structure.
- Handle missing attendance data gracefully.

US2 – Strong Course Identification
- Analyze student grades per course.
- Calculate performance levels dynamically.
- Identify courses where the student performs significantly above the student’s overall average.
- Highlight strong courses in the heatmap.
- Add tooltip/details with exact performance values.

US3 – Weak Course Identification
- Analyze student grades per course.
- Detect courses where performance is significantly below the student’s overall average.
- Mark weak courses visually in the heatmap.
- Display warning indicators for weak areas.

US4 – Performance Color Mapping
- Use a clear performance scale:
  Dark Green = Excellent
  Light Green = Good
  Yellow = Average
  Orange = Needs Improvement
  Red = Weak
- Add a visible legend.
- Keep color usage consistent.
- Do not rely only on color; add labels, icons, badges, or tooltips where useful.

US5 – Category Filtering
- Add filter controls above the heatmap.
- Allow switching between Attendance View and Grades View.
- Update the heatmap dynamically without page refresh.
- Keep the layout responsive.

US6 – Class Average Comparison
- Reuse existing class average data/logic if already available.
- Compare the student’s performance against class averages per course.
- Show comparison indicators inside the heatmap.
- Add tooltip/details with:
  student value,
  class average,
  difference percentage.
- Do not duplicate or replace the existing Comparison to Class Average feature.

US7 – Heatmap-Based Recommendations
- Analyze weak courses and attendance patterns from the heatmap data.
- Generate simple recommendations based only on Heatmap Analytics insights.
- Display them in a Recommendations panel below the heatmap.
- Do not build the full Smart Recommendations module yet.
- Do not duplicate or rewrite Risk Alerts.
- Recommendations should be generated from real/demo data, not hardcoded UI text.
